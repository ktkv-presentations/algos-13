Модульное программирование

метод разработки программного обеспечения, при котором программа разбивается на небольшие независимые части

Преимущества

Меньше повторений в коде
Упрощение логики программы
Облегчение отладки и тестирования

Модуль

отдельный файл в программе, содержащий функции, классы, переменные и другие объекты, которые можно использовать в других частях программы через импорт

В Python модуль — это любой файл с расширением .py

Пример модуля в Python

fibo.py 

def fib(n):
    a, b = 0, 1
    while a < n:
        print(a, end=' ')
        a, b = b, a+b
    print()

def fib2(n):
    result = []
    a, b = 0, 1
    while a < n:
        result.append(a)
        a, b = b, a+b
    return result
main.py 

import fibo

Типы модулей

Встроенные модули

Предоставляются вместе с интерпретатором

  • math — предоставляет математические функции
  • os — содержит функции для работы с операционной системой
  • random — используется для генерации случайных чисел

Пользовательские модули

Модули, которые создает пользователь или модули, которые предоставляют установленные библиотеки

  • from django.db import models
  • import fibo
  • import numpy as np

Встроенные модули

Метод поиска модулей

  1. Python сначала ищет модуль в текущем рабочем каталоге
  2. Затем, если модуль не найден, проверяются каталоги, указанные в переменной окружения PYTHONPATH
  3. В последнюю очередь Python обращается к стандартной библиотеке и каталогам, указанным в списке sys.path
  4. В указанной пользователем директории

import sys
sys.path.append('/path/to/my/modules')

Имя модуля и его использование

Каждый модуль имеет специальную переменную __name__ , которая содержит имя модуля. 


if __name__ == "__main__":
    print("Этот код выполняется, если модуль запущен напрямую")
else:
    print("Этот код выполняется, если модуль импортирован")

Если модуль запущен напрямую, __name__ будет равно __main__. Если модуль импортирован, его имя будет именем файла.

Хранение модулей

Модули хранятся рядом с главным файлов main.py или в папках внутри проекта 


/project
├── data_processing.py  # Модуль для обработки данных
├── user_interface.py   # Модуль для взаимодействия с пользователем
└── main.py             # Основной файл программы

    def __init__(self, radius):
        self.radius = radius

    def area(self):
        return PI * self.radius ** 2

Документация модулей

Автоматическая генерация документации из комментариев благодаря docstrings 


def add(a, b):
    """
    Функция для сложения двух чисел.
    :param a: Первое число
    :param b: Второе число
    :return: Сумма a и b
    """
    return a + b
Первая строка - описывает функцию
:param название: - описание аргумента
:return описание_возврата: - описание данных, которые возвращает модуль

Pip Installs Packages

система управления пакетами Python и их зависимостями

Установка pip

Обычно pip устанавливается вместе с Python. Проверить можно с помощью команды pip --version

Если pip не установлен - это можно сделать с помощью команды python -m ensurepip --upgrade

Основные команды pip

pip install имя_пакета

Установка нового пакета глобально

Также можно уточнить версию пакета

pip install requests==2.25.1
pip install --upgrade имя_пакета

Обновление пакета до последней версии

Основные команды pip

pip uninstall имя_пакета

Удаление пакета

pip list

список всех установленных пакетов и их версий

Виртульная среда venv

среда, которая позволяет изолировать зависимости проекта от глобальных установок Python

Создание виртуальной среды

python -m имя_окружения .папка_окружения python -m venv .venv

Активизация виртуальной среды

source имя_окружения/bin/activate

Установка пакетов виртуальной среде

pip install requests flask

Выход из виртуальной среды

deactivate
